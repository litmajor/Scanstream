## 🎯 TWO KEY RULES

### Rule 1: Skip Low-Confidence Trades
```
IF signal.confidence < 0.5
THEN return HOLD (don't enter)
```
- Empirical data: 44.7% WR vs 50.0% WR for high confidence
- Impact: Filters 37.6% of trades
- These are false signals masquerading as real ones

---

### Rule 2: Avoid Turbulent_chop Regime
```
IF regime == 'turbulent_chop'
THEN reduce position size 70% (or skip entirely)
```
- Empirical data: 44.8% WR vs 50.0% in consolidation
- Impact: Reduces position size in choppy markets
- Turbulent chop = noise mistaken for direction

## 📋 COPY-PASTE CODE

### Minimal Implementation (Add to VFMDPhysicsAgent.ts generateSignal())
```typescript
// After confidence is calculated, before final return statement
if (confidence < 0.5) {
  // Skip low-confidence trades entirely
  return { action: 'HOLD', confidence, target: 0, stop: 0, metadata };
}

// Check current regime
const regime = regimeClassifier.getCurrentRegime();
if (regime === 'turbulent_chop') {
  // Reduce position confidence by 70%
  metadata.positionMultiplier = 0.3;  // 30% of normal position size
}

// Continue with normal signal generation...
```

### Full Implementation with All Regimes
```typescript
// Regime-aware position multipliers
const regimeMultipliers: Record<string, number> = {
  'consolidation': 1.0,           // Stay full size (50% WR)
  'laminar_trend': 0.85,          // 85% position
  'accumulation': 0.8,            // 80% position  
  'breakout_transition': 0.6,     // 60% position (uncertain)
  'distribution': 0.4,            // 40% position (declining)
  'turbulent_chop': 0.3,          // 30% position (44.8% WR!)
};

// After confidence calculated:
if (confidence < 0.5) {
  return { action: 'HOLD', confidence, target: 0, stop: 0, metadata };
}

const regime = regimeClassifier.getCurrentRegime();
metadata.positionMultiplier = regimeMultipliers[regime] ?? 0.5;

// Continue with normal signal...
```
